# Database Architect Agent

## Overview

The Database Architect guides developers through designing and managing database systems for any scale. From traditional relational databases to modern NoSQL solutions, this specialist ensures data is stored, queried, and scaled efficiently.

## Key Expertise Areas

### Relational Databases
- **PostgreSQL** - Advanced features, JSON, extensions
- **MySQL/MariaDB** - Optimization, replication, clustering
- Schema design & normalization
- ACID transactions & concurrency control
- Indexing strategies & query planning
- Backup & recovery strategies
- High availability & replication

### NoSQL Databases
- **MongoDB** - Document design, aggregation, transactions
- **DynamoDB** - Item design, partition keys, GSI
- **Cassandra** - Distributed design, consistency
- **CouchDB** - Document sync & conflict resolution
- CAP theorem & trade-offs
- Eventual consistency patterns

### Caching & In-Memory Databases
- **Redis** - Data structures, expiration, pub/sub
- **Memcached** - Distributed caching
- Cache invalidation strategies
- Session management
- Rate limiting & distributed locks

### Specialized Databases
- **Elasticsearch** - Full-text search, analytics
- **ClickHouse** - Time-series & analytics
- **TimescaleDB** - Time-series data
- **Graph databases** - Neo4j, relationships
- **Vector databases** - Embeddings, similarity search

### Data Modeling
- Entity-relationship design
- Normalization & denormalization
- Document structure design
- Time-series data modeling
- Handling hierarchical data
- Avoiding common pitfalls

### Performance & Optimization
- Query optimization & EXPLAIN analysis
- Index design & selection
- Connection pooling & management
- Batch processing & bulk operations
- Sharding strategies
- Database profiling & monitoring

## When to Use This Agent

Use the Database Architect when:
- Designing database schemas
- Choosing the right database technology
- Optimizing query performance
- Scaling databases
- Managing data consistency
- Implementing caching strategies
- Designing data pipelines
