# /design-ux

Design intuitive and beautiful interfaces for your plugin with expert guidance from the UX/UI Design Expert.

## Usage

```
/design-ux [plugin-name] [--research] [--wireframe] [--prototype] [--test]
```

## Examples

### Start UX design process
```
/design-ux my-plugin
```

### Conduct user research
```
/design-ux my-plugin --research
```

### Create wireframes
```
/design-ux my-plugin --wireframe
```

### Build prototypes
```
/design-ux my-plugin --prototype
```

### Run usability tests
```
/design-ux my-plugin --test
```

## Design Process

### Phase 1: Research (--research)

Understand your users:
- User interviews
- Persona creation
- Pain point analysis
- Use case mapping
- Competitive analysis
- Accessibility needs

Output:
- User personas
- Use case diagrams
- Pain point analysis
- Accessibility audit
- Competitive report

### Phase 2: Wireframing (--wireframe)

Plan your interface:
- Information architecture
- Layout planning
- Navigation design
- Component identification
- User flows
- Interaction patterns

Output:
- Wireframe sketches
- User flow diagrams
- Component inventory
- Navigation map
- Interaction matrix

### Phase 3: Design (default)

Create visual design:
- Visual hierarchy
- Typography
- Color palette
- Component design
- Icon design
- Animation planning

Output:
- Design mockups
- Design system
- Component library
- Style guide
- Animation specs

### Phase 4: Prototyping (--prototype)

Build interactive prototypes:
- Interactive flows
- Micro-interactions
- Animation effects
- State management
- Responsive design
- Accessibility features

Output:
- Interactive prototype
- Prototype specifications
- Responsive design guide
- Accessibility checklist

### Phase 5: Testing (--test)

Validate your design:
- Usability testing
- A/B testing
- Accessibility testing
- Performance testing
- User feedback
- Analytics

Output:
- Test results
- User feedback report
- Improvement recommendations
- Accessibility report
- Analytics insights

## Full Design Workflow

```
/design-ux my-plugin --research --wireframe --prototype --test
```

Combines all phases for complete UX design process.

## Accessibility Checklist

Ensure WCAG compliance:
- [ ] Color contrast (4.5:1 minimum)
- [ ] Keyboard navigation
- [ ] Screen reader support
- [ ] Alternative text for images
- [ ] Focus indicators
- [ ] Motion alternatives
- [ ] Text size flexibility
- [ ] Error message clarity

## Tips for Great Design

1. **User-Centered** - Always design for users
2. **Simple** - Simplicity reduces cognitive load
3. **Consistent** - Consistency builds familiarity
4. **Accessible** - Design for everyone
5. **Testable** - Validate assumptions
6. **Iterable** - Improve based on feedback
7. **Documented** - Document your decisions
